Docker Network Drivers
-----------------------------
1. Bridge: default(Docker0)
2. Host: isolation between the container and the docker host
3. None: completely isolates a container from the host and other containers
4. Overlay: overlay containers multiple docker daemons together
5. Ipvlan: provides full control over both IPV4 and IPV6 addressing
6. Macvlan: assign a MAC address to container
7. Userdefind: a custom network ^mHG3v8LW

Bridge Driver:
----------------
1. containers connected to the same bridge network can communicate
2. a bridge network isolates its containers from other containers on other bridges
3. bridge networks apply within the same Docker host only
4. A container can connect to multiple bridge network at the same time
